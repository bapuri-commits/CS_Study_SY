- **HANDOFF는 날짜별 누적, 필요 시 통합**: 스냅샷 덮어쓰기가 아닌 이력 보존
- **세션 컨텍스트를 HANDOFF에 포함**: 대화 요약도 기록 (토큰 제한 없음)
- **읽기 트리거는 기기 변경 시에만**: 매 세션 자동 읽기 안 함 (토큰 절약)
- **워크스페이스 핸드오프 위치**: `.workspace/handoff/` — CS_Study_SY 루트 Git 레포가 추적
- **The Record와 완전 별개**: 목적이 다름 (AI 연속성 vs 인간 성찰)
- **프로젝트별 규칙은 해당 프로젝트 스코프로 제한**: alwaysApply 남용 문제 해결
- **원본 대화 기록은 VPS에 저장 예정**: Contabo VPS 세팅 후 구현
- **프로젝트 add/remove 자동화**: 수동으로 4곳 수정하던 것을 sync.py 한 줄로 해결

## 미논의 사항: 핸드오프 커밋/푸시 워크플로우
이번 세션에서 핸드오프를 어떻게 git에 반영할지 구체적 논의가 없었음. 현재 상태:
- **핸드오프 파일은 각 프로젝트 Git 레포에 포함** — `docs/handoff/`가 프로젝트 디렉토리 안에 있으므로 자연스럽게 추적됨
- **워크스페이스 핸드오프는 CS_Study_SY 루트 레포에 포함** — `.workspace/handoff/`
- **커밋/푸시는 현재 수동** — 핸드오프 작성 후 사용자가 직접 또는 AI에게 요청
- **자동화 가능성**: sync.py에 `push` 서브커맨드 추가, 또는 핸드오프 쓰기 트리거에 "커밋까지 할까요?" 제안
- **주의**: 핸드오프만 별도 커밋하면 커밋 히스토리가 지저분해질 수 있음 — 작업 커밋에 핸드오프를 같이 포함하는 게 깔끔할 수 있음
- 데스크탑에서 이어서 논의/결정 필요
